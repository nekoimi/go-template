### 数据库gorm必须的优化

1、数据库的ID使用雪花算法（不考虑多机部署），需要考虑前端js精度丢失问题，json返回最好返回string

2、整个项目的时区优化，项目将使用docker部署，使用环境变量配置时区，数据库返回的时间需要格式化（年-月-日 时-分-秒），并且采用正确的时区显示

### 高优先级推荐添加（强烈建议尽快加上，让模板更生产可用）

1. **OpenAPI / Swagger 文档自动生成**
    - 库：`github.com/swaggo/gin-swagger` + `github.com/swaggo/files` + `github.com/swaggo/swag`
    - 好处：接口文档自动生成，`/swagger/index.html` 可直接访问，团队协作和前端对接极度友好。
    - 实现：在 handler 上加注释，启动时自动生成。
    - 建议加入 `internal/handler/v1` 的每个 handler。

2. **Rate Limiting（限流）**
    - 推荐库：`github.com/ulule/limiter/v3` 或 `golang.org/x/time/rate` + Gin middleware。
    - 全局 + 按 IP / 用户 / 路径限流，防止滥用，尤其对 WebSocket 和文件上传很重要。

3. **优雅关闭（Graceful Shutdown）增强 + 信号处理**
    - 已经有了基础，建议完善：捕获 SIGTERM/SIGINT，等待正在处理的请求/WebSocket 完成，关闭 cron、数据库连接、storage 等。
    - 加上 context 超时控制。

4. **统一错误处理 + 自定义错误码系统**
    - 在 `internal/pkg/errors` 或 `pkg/response` 中定义 `AppError` 结构体（Code、Message、Details）。
    - Gin 中间件统一 recover 并返回标准 JSON 格式。
    - 便于前端和监控系统处理。

5. **请求日志中间件（结构化）**
    - 用 Zap 记录每个 HTTP 请求（method、path、status、latency、client IP、request ID）。
    - 与 Prometheus Metrics 配合使用。

### 中优先级推荐（根据项目规模选择性添加）

6. **Caching 层（Redis）**
    - 支持本地缓存 + Redis。
    - 类似 Storage 的策略模式，便于后续扩展。
    - 特别适合热点数据、会话、限流等。

7. **邮件 / 通知系统**
    - SMTP 或集成 SendGrid / 阿里云邮件 等。
    - 模板化邮件（HTML），异步发送（结合 scheduler 或 worker）。

8. **任务队列（异步任务）**
    - 当前 cron 适合定时，建议加 Asynq（Redis 基于）或 RabbitMQ，支持延迟任务、重试、优先级。
    - 与 scheduler 配合使用。

9. **RBAC / 权限控制**
    - 基于 JWT claims 的简单角色权限（admin/user），或 Casbin（策略灵活）。
    - 中间件检查权限。

10. **请求追踪（Tracing）**
    - OpenTelemetry（与 Prometheus 配合），支持 Jaeger / Tempo 可视化。
    - 对分布式调用和 WebSocket 调试很有帮助。

11. **数据库健康检查 & Readiness/Liveness Probe**
    - `/health`、` /ready` 接口，包含 DB、Redis、MinIO 等组件检查。
    - Kubernetes / Docker 部署必备。

### 低优先级 / 优化方向（视情况）

- **依赖注入框架**：当前手动 NewXXX 即可。小项目够用；项目变大可加 `google/wire`。
- **测试模板**：提供 `internal/service` 的单元测试示例 + testcontainers 集成测试模板。
- **CI/CD**：GitHub Actions 示例（lint、test、build、docker push）。
- **Lint & 格式化**：`.golangci.yml` 配置 + Makefile `make lint`。
- **版本管理 & Semantic Release**：CHANGELOG 自动生成。
- **热重载（开发友好）**：Air 或 CompileDaemon。
- **gRPC 支持**（可选）：如果未来需要高性能内部服务。
- **国际化（i18n）**：如果有多语言需求。
- **审计日志**：记录敏感操作（谁在什么时候做了什么）。

### 当前模板的优化建议（不增加新功能）

- **目录微调**：把 `pkg/` 下的通用工具再细分（`pkg/errors`、`pkg/middleware`、`pkg/utils`）。
- **配置文件**：支持多环境合并（dev / test / prod），用 viper 的 `MergeConfig`。
- **Storage & Metrics**：在文件上传成功时增加 Prometheus counter；在 WebSocket manager 中暴露连接数 gauge。
- **安全性**：CORS 中间件严格配置；文件上传增加 MIME 类型白名单 + 大小限制 + 防路径穿越。
- **文档**：在 README.md 中增加「如何扩展新 Storage 驱动」「如何添加新指标」等小节。
- **Go 版本**：在 go.mod 中锁定 Go 1.23+ 或更高，充分利用最新特性。
