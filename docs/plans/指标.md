采用业界标准方案：**Prometheus + client_golang**（官方推荐），结合 Gin 使用中间件自动采集 HTTP 请求指标。同时提供自定义指标收集器（数据库连接池、Goroutine、内存等），并暴露 `/metrics` 接口，方便 Prometheus 抓取 + Grafana 可视化。

这个实现**通用、可扩展**，对现有 HTTP、数据库、WebSocket、定时任务、文件存储等模块零侵入或最小改动。

### 核心设计（推荐实践）

- 使用 **自定义 Registry**（避免污染全局默认 registry，便于测试和多服务）。
- **Gin Middleware**：自动记录 HTTP 请求（method、path、status、duration、size）。
    - 使用 `prometheus/client_golang` 官方 middleware 或简单自定义（推荐官方 promhttp 结合 Gin WrapH）。
- **Collector**：实现 `prometheus.Collector` 接口，动态采集 DB 连接池、系统指标。
- **统一注册**：在启动时一次性注册所有指标。
- **标签（Labels）**：低基数（method、status_code、path 等），避免高基数（如 user_id）导致爆炸。
- **扩展性**：以后加 WebSocket 连接数、文件上传计数、定时任务执行次数，只需在对应模块注册新指标即可。

### 常用指标示例（模板会自动提供）

- **HTTP**：`goscaffold_http_requests_total`、`goscaffold_http_request_duration_seconds`（histogram）、错误率等。
- **数据库**：连接池大小、空闲连接、等待时间、在用连接。
- **系统**：Goroutine 数量、堆内存使用、GC 暂停时间、CPU 等（Go 自带 + 自定义）。
- **业务**：可后续轻松添加（如上传成功计数、登录失败次数）。

### Prometheus + Grafana 集成（开发友好）

`docker-compose.yml` 会新增：
- prometheus 服务（scrape `app:8080/metrics`）
- grafana 服务（预置 Dashboard 示例）

访问：
- Metrics：`http://localhost:8080/metrics`
- Prometheus：`http://localhost:9090`
- Grafana：`http://localhost:3000`（默认 admin/admin）

### 使用方式

1. 在 `cmd/server/main.go` 中初始化 `metrics.Registry` 并注册 middleware。
2. Gin 路由中添加：`router.GET(cfg.Metrics.Path, gin.WrapH(promhttp.HandlerFor(registry, promhttp.HandlerOpts{})))`
3. Middleware 放在路由注册后（或全局 Use）。
4. 自定义 Collector 在启动时注册：`registry.MustRegister(dbCollector)`
