---

一个轻量、实用、生产就绪的 **Go 后端模板项目**，专为中小型项目设计。

采用 **Gin** + **GORM** + **PostgreSQL** 技术栈，结构扁平清晰，适合快速开发 MVP 或内部工具。避免过度分层，降低学习和维护成本，同时保留良好的可扩展性。

### 技术栈

- **Web 框架**：Gin (v1.10+)
- **ORM**：GORM v2（PostgreSQL 驱动）
- **配置管理**：Viper（支持 YAML + 环境变量覆盖）
- **日志**：Zap（结构化日志）
- **认证**：JWT (golang-jwt) + bcrypt 密码哈希
- **参数验证**：go-playground/validator/v10
- **数据库迁移**：golang-migrate（推荐版本化 SQL 迁移，生产环境强烈建议使用，而非 GORM AutoMigrate）
- **其他**：优雅关闭、统一响应格式、健康检查、中间件（CORS、Logger、Recovery、JWT）

**Go 版本要求**：Go 1.23+

### 项目目录结构

```
go-project-template/
├── cmd/
│   └── server/
│       └── main.go                  # 程序入口：配置加载、依赖初始化、Gin 启动
├── internal/
│   ├── config/                      # 配置结构体与加载
│   │   ├── config.go
│   │   └── loader.go
│   ├── handler/                     # HTTP 处理层
│   │   ├── v1/                      # API 版本（推荐使用 v1）
│   │   │   ├── auth.go
│   │   │   └── user.go
│   │   └── middleware/              # 中间件
│   ├── model/                       # GORM 数据模型
│   ├── repository/                  # 数据库操作实现
│   ├── service/                     # 业务逻辑层
│   ├── dto/                         # 请求/响应 DTO（可选）
│   └── infrastructure/              # 基础设施
│       ├── database/
│       │   └── postgres.go
│       └── logger/
│           └── zap.go
├── migrations/                      # 数据库迁移脚本（.up.sql / .down.sql）
├── configs/                         # 配置模板
│   ├── config.dev.yaml
│   ├── config.prod.yaml
│   └── config.test.yaml
├── deployments/
│   ├── Dockerfile
│   └── docker-compose.yml
├── .env.example
├── go.mod
├── Makefile
├── README.md
└── docker-compose.yml
```

**设计理念**：
- 小项目优先：业务代码直接放在 `internal/` 下（handler/service/repository/model），无多余 `app/` 或 `domain/` 层。
- 关注点分离：Handler 只负责 HTTP、Service 处理业务规则、Repository 只做数据访问。
- 易于扩展：未来项目变大时，可轻松拆分成更严格的 Clean Architecture。

### 快速开始

#### 1. 初始化项目

```bash
git clone https://github.com/nekoimi/go-project-template.git
cd go-project-template
go mod tidy
```

#### 2. 配置

复制配置模板：

```bash
cp configs/config.dev.yaml configs/config.yaml
cp .env.example .env
```

修改 `configs/config.yaml` 中的数据库、JWT 等信息。

#### 3. 数据库迁移

安装 migrate CLI（推荐）：

```bash
go install -tags 'postgres' github.com/golang-migrate/migrate/v4/cmd/migrate@latest
```

运行迁移（推荐在 Makefile 中执行）：

```bash
make migrate-up
```

#### 4. 启动服务

```bash
make run
# 或直接
go run cmd/server/main.go
```

服务默认启动在 `:8080`，访问 `http://localhost:8080/health` 测试。

#### 5. 使用 Docker（推荐本地开发）

```bash
make docker-up     # 启动 postgres + app
make docker-down
```

### 核心功能示例

- **认证**：`/v1/auth/register`、`/v1/auth/login`（返回 JWT）
- **JWT 中间件**：保护需要登录的路由
- **统一响应**：`{ "code": 200, "message": "success", "data": {...} }`
- **健康检查**：`/health`
- **结构化日志**：请求耗时、状态码、trace 等

### 配置说明

Viper 支持以下方式（优先级：环境变量 > YAML > 默认值）：

- `configs/config.dev.yaml`
- 环境变量（自动绑定，例如 `DATABASE_HOST`）
- `.env` 文件（可选）

关键配置字段包括：
- `server.port`、`server.mode`（debug/release）
- `database`（host、port、user、password、dbname、连接池）
- `jwt.secret`、`jwt.expire_hours`

### 数据库迁移最佳实践

- 使用 **golang-migrate** 进行版本化管理
- 迁移文件存放在 `migrations/` 目录
- 命名规范：`000001_create_users_table.up.sql` / `.down.sql`
- **生产环境禁止** 使用 GORM `AutoMigrate`，仅在本地开发时可选使用
- 迁移脚本应包含完整的 CREATE TABLE、INDEX、CONSTRAINT 等

### 开发规范建议

1. **统一响应**：所有接口使用 `internal/pkg/response` 中的 Success / Error 函数
2. **错误处理**：使用自定义错误码 + Zap 记录详细错误
3. **参数验证**：在 DTO struct 上使用 validator tag（`binding:"required,email"`）
4. **依赖注入**：小项目采用手动 NewXXX 方式（简单清晰）；后期可引入 wire
5. **测试**：单元测试覆盖 Service，集成测试使用 testcontainers 或 docker-compose
6. **日志**：所有业务日志通过注入的 Logger，不要使用全局变量
